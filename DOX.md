# DOX Methodology

This repository follows the DOX methodology for continuous documentation, ensuring that AI agents and human developers maintain shared context across features, current state, and long-term plans.

## DOX Structure
The documentation is organized in the `docs/` directory:

- [Project Brief](docs/project-brief.md): What this is, who it's for, and the functional specs.
- [Current State](docs/current-state.md): What is explicitly implemented right now.
- [Roadmap](docs/roadmap.md): Wishlist, bucket-list ideas, and future phases.
- [Architecture](docs/architecture.md): Frontend, backend, data, and deployment design.
- [Open Questions](docs/open-questions.md): Pending decisions.
- [Implementation Notes](docs/implementation-notes.md): Gotchas and engineering boundaries.

## Agent Guidelines
AI Agents must read this file and `AGENTS.md` before making sweeping changes. Pay special attention to the nested `AGENTS.md` hierarchy used throughout this repository.
